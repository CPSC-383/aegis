## Installation

<Steps>

### Download the assignment

Download the ZIP file and unzip it with your preferred method.
If successful, a new directory called aegis should have been created.

### Navigate to the assignment directory

```bash
cd aegis
```

### Setting up the assignment

Aegis requires a set of Python packages to function. Installing these in a virtual environment
ensures there are no conflicts with other Python projects or system-wide packages.

Run the setup script:

!!! ADD TABS HERE 

If the script was successful, a virtual environment named `.venv` should have been created, the required packages
installed within it and the client for your platform should be in the `client` folder.

Steps do to it manually if the setup script failed:

!!! ADD TABS HERE 

Unzip your platform client in the client folder with your preferred method.

### Confirm Installation

To confirm that Aegis installed properly, follow the steps outlined <a href="/getting-started/aegis/running-aegis">here</a>.

</Steps>

<Aside variant="tip" title="Virtual Environment">
  Deactivate the virtual environment once you're done working on the assignment:
  
  ```bash
  deactivate
  ```

  You must reactivate the virtual environment every time you work on the assignment. If you're not inside the environment,
  you may encounter missing dependency errors.
</Aside>


## File Structure

The Aegis client uses a specific structure for agents and worlds. Be sure to follow
these guidelines so the client can find and run the necessary files correctly.

### Agent Structure

<FileTree>
  <FileTreeItem name="aegis" type="folder">
    <FileTreeItem name="src" type="folder">
      <FileTreeItem name="agents" type="folder">
        <FileTreeItem name="example_agent" type="folder">
          <FileTreeItem name="example_agent.py" />
          <FileTreeItem name="main.py" />
        </FileTreeItem>
      </FileTreeItem>
    </FileTreeItem>
  </FileTreeItem>
</FileTree>

<ul>
  <li>Place new agents in the `src/agents` directory.</li>
  <li>Agents must include a `main.py` file.</li>
</ul>

### World Structure

<FileTree>
  <FileTreeItem name="aegis" type="folder">
    <FileTreeItem name="worlds" type="folder">
      <FileTreeItem name="ExampleWorld.world" />
    </FileTreeItem>
  </FileTreeItem>
</FileTree>

<ul>
  <li>Place new worlds in the `worlds` directory.</li>
  <li>All world files must end with `.world`.</li>
</ul>
