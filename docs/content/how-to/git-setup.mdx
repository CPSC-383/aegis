## Overview

This guide provides a step-by-step process for setting up a Git repository for your group assignment. Only one person
in the group needs to create the repository and the `.gitignore` file.

## Prerequisites: Installing Git

Ensure Git is installed on your system using:

```bash
git --version
```

If it is not installed, follow the steps below:

<Tabs defaultValue="mac">

<TabsList>

<TabsTrigger value="mac">Mac</TabsTrigger>
<TabsTrigger value="linux">Linux</TabsTrigger>
<TabsTrigger value="windows">Windows</TabsTrigger>

</TabsList>

<TabsContent value="mac">

Install Git using Homebrew:

```bash
brew install git
```

If you don't have Homebrew, first install it from [brew.sh](https://brew.sh/)

</TabsContent>

<TabsContent value="linux">

Install Git using the package manager included with your Linux distribution.

</TabsContent>

<TabsContent value="windows">

Download Git from [here](https://git-scm.com/downloads/win).

</TabsContent>

</Tabs>

## Initializing Git in an Existing Project

<Steps>

### Navigate to the Aegis Directory

Open a terminal and change into the Aegis directory.

```bash
cd path/to/aegis/directory
```

### Initialize the Git Repo

Initialize Git in the Aegis directory.

```bash
git init
```

### Create a New Repository

<Tabs defaultValue="github">

<TabsList>

<TabsTrigger value="github">Github</TabsTrigger>
<TabsTrigger value="gitlab">Gitlab</TabsTrigger>

</TabsList>

<TabsContent value="github">

1. Go to [GitHub](https://github.com) and log in.
2. Click the "New" button to create a new repository.
3. Enter a repository name, choose private for the visibility, and click "Create Repository".

</TabsContent>

<TabsContent value="gitlab">

1. Go to [Gitlab](https://csgit.ucalgary.ca/) and log in with your IT account.
2. Click the "+" button and then click "New project/repository".
3. Create a blank project.
3. Enter a project name, choose private for the visibility, and click "Create Project".

</TabsContent>

</Tabs>

## Creating a .gitignore File

<Admonition variant="caution" title="Important Placement">
The `.gitignore` file must be in the root directory of your project to work correctly.
</Admonition>

A `.gitignore` file tells Git which files to ignore.

<Steps>

### Create the .gitignore File

<Tabs defaultValue="mac/linux">

<TabsList>

<TabsTrigger value="mac/linux">Mac/Linux</TabsTrigger>
<TabsTrigger value="windows">Windows</TabsTrigger>

</TabsList>

<TabsContent value="mac/linux">

Creating a file in the command line:

```bash
touch .gitignore
```

or you can use the file explorer.

</TabsContent>

<TabsContent value="windows">

Creating a file in the command line:

```bash
echo "" > .gitignore
```

or you can use the file explorer.

</TabsContent>

</Tabs>

### Add the Following Content 

```gitignore

client
.venv
venv
WorldInfoFile.out
*__pycache__*

# Editor directories and files
.DS_Store
.vscode/*
!.vscode/extensions.json
.idea
*.suo *.ntvs*
*.njsproj
*.sln
*.sw?
```

## Essential Git Workflow Commands

### Pull Latest Changes 

<Admonition variant="tip" title="Collaboration Best Practices">
Minimize merge conflicts and maintain smooth team collaboration by following these key Git workflow principles:

- Always pull changes before starting work to ensure you have the latest project state.
- Communicate with your team about who is working on specific files or features.
- Pull changes from the remote repository immediately before pushing your commits.
- Break your work into small commits that allow you to easily revert changes if something breaks. This provides a clear rollback path and reduces the risk of losing significant work.

</Admonition>

```bash
git pull
```

### Pushing Your Changes

```bash
git add .
git commit -m "Descriptive commit message"
git push
```

### Checking Repository Status

```bash
git status
```
