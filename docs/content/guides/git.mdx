## Overview

This guide provides a step-by-step process for setting up a Git repository for your group assignment.
Only one person in the group needs to create the repository and the `.gitignore` file.

<Callout title="Version Control Recommended">
Setting up a Git repository is optional, but highly recommended for managing code collaboratively.
</Callout>

## Installing Git

Ensure Git is installed on your system using:

```bash
git --version
```

If Git is installed, youâ€™ll see output similar to:

```bash
git version x.xx.x
```

If you receive an error or no output, follow the steps below:

<Tabs items={['Mac', 'Linux', 'Windows']}>

<Tab value="Mac">

Install git using Homebrew:

```bash
brew install git
```

If you don't have Homebrew, first install it from [brew.sh](https://brew.sh/)

</Tab>

<Tab value="Linux">

Install git using the package manager included with your Linux distribution.

</Tab>

<Tab value="Windows">

Download git from [here](https://git-scm.com/downloads/win).

</Tab>

</Tabs>

## Initializing Git in an Existing Project

<Steps>


### Navigate to the Aegis Directory

Open a terminal and change into the Aegis directory

```bash
cd path/to/aegis/directory
```

### Initialize the Git Repo

Initialize Git in the Aegis directory

```bash
git init
```

### Create a New Repository

<Tabs items={['Github', 'Gitlab']}>

<Tab value="Github">

1. Go to [GitHub](https://github.com) and log in.
2. Click the "New" button to create a new repository.
3. Enter a repository name, choose private for the visibility, and click "Create Repository".

</Tab>

<Tab value="Gitlab">

1. Go to [Gitlab](https://csgit.ucalgary.ca/) and log in with your IT account.
2. Click the "+" button and then click "New project/repository".
3. Create a blank project.
4. Enter a project name, choose private for the visibility, and click "Create Project".

</Tab>

</Tabs>

### Add a Remote Repository

Link your aegis repository to the newly created remote repository.

<Tabs items={['Github', 'Gitlab']}>

<Tab value="Github">

```bash
git remote add origin https://github.com/yourusername/your-repo.git
```

</Tab>

<Tab value="Gitlab">

```bash
git remote add origin https://csgit.ucalgary.ca/yourusename/your-repo.git
```

</Tab>

</Tabs>

</Steps>

## Creating a .gitignore file

<Callout type="warning" title="File Placement">

The `.gitignore` file must be in the root of your project to work correctly.

</Callout>

A `.gitignore` file tells git which files to ignore.

<Steps>

### Create the .gitignore file

<Tabs items={['Mac/Linux', 'Windows']}>

<Tab value="Mac/Linux">

Creating a file in the command line:

```bash
touch .gitignore
```

or you can use the file explorer.

</Tab>

<Tab value="Windows">

Creating a file in the command line:

```bash
echo "" > .gitignore
```

or you can use the file explorer.

</Tab>

</Tabs>

### Add the Following Content

```txt
client
.venv
venv
*__pycache__*
 
# Editor directories and files
.DS_Store
.vscode/*
!.vscode/extensions.json
.idea
*.suo *.ntvs*
*.njsproj
*.sln
*.sw?
```

### Initial Commit and Push

<Steps>

### Stage and Commit Files

```bash
git add .
git commit -m "Initial Commit"
```

### Push to the Repository

```bash
git branch -M main
git push -u origin main
```

</Steps>
